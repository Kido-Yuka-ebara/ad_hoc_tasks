# 成果物1: 情報の整理と棚卸し

## 1. ヒアリング項目と回答

### 1.1. プレゼンの対象・目的について

- **対象者:**
  - 自部署（生産プロセス技術開発課）に配属された新入社員1名。
- **最終的なゴール（当初の希望）:**
  - 担当業務（構内物流改善）について知ってもらう。
  - 関連業務で質問できる人間として認知してもらう。
  - 自身の仕事の進め方（一例）を伝え、本人の業務の参考にしてもらう。
  - 部署の文化（テーマアップ型）を理解し、自身のテーマ設定について考え始めてもらう。

### 1.2. 業務内容について

- **部署名と役割:**
  - **部署名:** 生産プロセス革新・品質保証統括部 生産革新技術部 生産プロセス技術開発課
  - **役割:** 製造業における生産プロセスを改善し、生産性を向上させる活動の推進。
- **紹介する担当業務:**
  - 構内物流改善の取り組み。
- **具体的な作業内容:**
  - 会社の仕組みを再設計するため、関係者との定例MTGを重ね、設計を固めている。
- **業務のやりがい:**
  - 自身の思想が「仕組み」として会社に残り、自分がいなくなってもその考えが生き続けると実感できた時。
  - 背景：前職で「やったことが残らない」と感じた経験が、現職への転職動機の一つ。
- **業務の難しさ・教訓:**
  - **難しさ:** 初期の計画立案。目的、背景、やり方を整理せずに進めると、手戻りが発生する。
  - **実体験:** 自身も当初、具体的なツール導入（課題解決）から始めようとしたが、全社的な仕組み・ルール作りを求められ、計画の変更を余儀なくされた。
  - **教訓:** MBO（目標管理制度）の評価が賞与や昇進に大きく影響するため、業務の進め方によっては当初の目標が達成できなくなる可能性がある。中間成果物によってフローが分岐しうることを初期段階で把握しておくことが重要。

### 1.3. その他

- **1日のスケジュール:**
  - 8:00始業、12:00昼食、16:30定時終業。
  - 定例MTG（週1回×4件：構内物流チーム、課長1on1、課会、別タスクチーム）が不定期な日時で開催される。
  - MTGの際にはアジェンダ作成と議事録作成も行う。
- **新入社員へのメッセージ:**
  - 最初はOJTリーダーの指示通りに進めれば良い。
  - 業務を経験する中で生まれる「こうしたい」という意志こそが、自身の信念や価値観の表れ。
  - その意志を軸に仕事を進めていけば、仕事は楽しくなる。

### 1.4. 追加ヒアリングによる深掘り

- **テーマ発見のきっかけ（課題の具体例）:**
  - 研修での工場見学時、長時間放置されている工程（モノの滞留）が非常に多いと感じたことが起点。
  - 部署の本質的な業務は「工場のムダをなくす」こと。理想は「原料が瞬時に製品になる」状態。
  - 現実的なアプローチとして、まず「モノが止まっている時間（滞留時間）」を減らすことに着目。
  - 個人的なセオリーとして、まず「モノ」の動きを最適化し、その後に「人」や「設備」の最適化に着手すべきだと考えている。
  - **現状のテーマアップ方法:** 定量的な課題把握の仕組みが未整備なため、メンバー個人の問題意識やシーズ（技術起点）がメインとなっている。

- **「良い計画」の要件:**
  - 根幹は「早い段階で、最終的な完成形をいかに解像度高くイメージし、定義できているか」。
  - **思考の順序:** Why（なぜ）→ What（何を）→ How（どうやって）の順で具体化する。特に最初の「Why」の明確化が最重要。
  - **計画の必須要素:**
    - 背景
    - 目的 (Why)
    - スコープ（やること・やらないこと）とその重要度
    - 関係者とそれぞれの権限
  - **計画策定後のプロセス:** 上記要素が固まって初めて、ロードマップ作成 → 工数見積もり → スケジュール策定 → 実行、というステップに進むことができる。

## 2. 結論：資料作成における3つのゴール設定

上記のヒアリング結果から、この資料が達成すべき最終的なゴールを以下の3つに具体化・設定する。

- **【ゴール1：道標としての認知】**
  - **目指す状態:** 新入社員が課題設定のヒントを得、今後改善について「何か知りたくなったら、まずはこの人に相談しよう」と思える状態を作る。
  - **期待する行動:** 業務上、関連する疑問やアイデアが浮かんだ際に、実際に質問しに来る。

- **【ゴール2：働き方の羅針盤の提供】**
  - **目指す状態:** 部署の文化（テーマアップ型）と、仕事の進め方の一例（計画立案の重要性、MBOとの関連など）を理解し、「自分はどう動けばいいか」の初期イメージが湧いている状態を作る。
  - **期待する行動:** OJTリーダーの指示をこなす中で、教わった内容と実務を結びつけて考え始める（例：「この作業が計画立案の部分か」「この成果はMBOにどう繋がるのか」）。

- **【ゴール3：主体性の発火】**
  - **目指す状態:** 「将来的には自分も、こんな風に課題を見つけて仕組みづくりに挑戦してみたい」と、当事者意識や将来へのモチベーションが芽生えている状態を作る。
  - **期待する行動:** 日々の業務の中で、「自分ならどんなテーマをやりたいか」「このプロセスはもっと良くならないか」といった問いを頭の片隅で考え始める。
