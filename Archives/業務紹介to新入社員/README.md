# README

## 概要

この一連のファイルは、新入社員向けに自身の業務内容を紹介する資料を作成するための手順と成果物をまとめたものです。

誰でも同じ手順で、質の高い紹介資料を作成できるよう、テンプレートとして活用することを目的としています。各ステップで作成される成果物は、手順がわかるようにナンバーが振られています。

## 作業フローと成果物一覧

- **01_情報の整理と棚卸し.md**
  - **目的:** 資料の骨子となる情報を洗い出し、整理する。
  - **内容:** ヒアリングシートに基づき、資料に盛り込むべき情報を構造化し、資料作成の最終的なゴールを明確に定義する。

- **02_資料構成の検討.md**
  - **目的:** 整理した情報を、最も伝わりやすい順番に並べ、資料全体の構成を決定する。
  - **内容:** 設定したゴールを達成するためのストーリーライン（目次）を作成する。

- **03-1_スライド構成.md**
  - **目的:** 02_資料構成の検討.mdを基に、スライドの大まかな構成と内容をMarkdown形式で作成する。
  - **内容:** 各スライドの見出し、本文、箇条書き、図表の配置イメージなどを、Markdown形式で記述する。デザインよりも内容の整理に集中する段階。

- **03-2_Marpスライド作成** (成果物は .md → .html/.pdf など)
  - **目的:** Markdown原稿をMarp形式に変換し、プレゼンテーション資料として完成させる。
  - **内容:** Marpの記法（`---`によるスライド区切り、テーマ設定など）を適用し、視覚的に整ったスライドを生成する。

- **04_レビューとフィードバック** (成果物は議事録やメモなど)
  - **目的:** 作成したドラフトを客観的に見直し、改善点を見つける。
  - **内容:** 第三者からのフィードバックや、自身での見直し結果を記録する。

- **05_修正と最終化** (成果物は完成版資料)
  - **目的:** フィードバックを反映し、資料を完成させる。
  - **内容:** 推敲、誤字脱字の修正、デザインの調整を行い、最終版の資料を完成させる。
