# 成果物0-2: デジボに関する所感

## 1. 適用可能な現場条件

デジボが有効に機能する現場条件は以下の通り:

- **管理者が常駐:** ホワイトボードをメインで管理する人が常にデスク作業を行う職種
- **単一管理者:** ホワイトボードを触る人が現場側で誰もいない

→ これらの条件を満たす現場ではメリットがある

## 2. 主な懸念事項

### 2.1 ツールとしての位置づけ

- **置き換えでしかない:** ログが残るのは良いが、あくまでアナログからデジタルへの置き換え
- **軽減効果は限定的:** 「人員配置調整作業」「人員配置可視化作業」がやや軽減される程度
- **ノーコード以外のメリット不明:** 直感的操作以外の差別化要素が不明瞭

### 2.2 データ連携の制約

- **データ連携不可（推定）:** ツール内に囲い込むスタイルのため、応用性は少ない
- **インプット側の不明点:** 人事データベースとの連携の自由度が不明瞭。結局ネームプレートを作る手間は残る可能性
- **アウトプット側の不明点:** データをツール外にリアルタイム連携できるのか不明

→ **要確認事項:** データのインプット/アウトプット連携の実現性

### 2.3 活用シーンの限定性

- **人員配置に特化:** 製品の動きを管理するようなホワイトボードの使用シチュエーションには使えない
- **活用シーンがかなり限定:** 汎用性が低い

## 3. 代替案の検討余地

### 3.1 汎用性の観点

- **応用性の高いツール:** もっと応用性の高いツールの方が良いのではないか
- **他ツールとの比較:** デジボであえてやるメリットがあるのか疑問

### 3.2 デジボのメリット（限定的）

以下のメリットは存在するものの、差別化要素としては弱い:

- 蓄積データを使った配置の自動化
- 分析機能
- 作業者スキル推定のプラットフォーム提供

→ これらの活用はできそうだが、**「あえてデジボでやるメリット」は不明瞭**

## 4. 総合所見

| 評価項目 | 評価 | コメント |
| :--- | :--- | :--- |
| **適用現場** | △ | 条件が限定的（単一管理者、常駐デスクワーク） |
| **効果** | △ | 置き換え効果のみ、軽減効果は限定的 |
| **データ連携** | × | インプット/アウトプット連携が不明瞭 |
| **汎用性** | × | 人員配置に特化、活用シーンが限定的 |
| **差別化** | × | あえてデジボを選ぶメリットが不明瞭 |

## 5. 導入前に確認すべき重要事項

1. **データ連携（アウトプット）:** ツール外へのリアルタイムデータ連携は可能か？
2. **データ連携（インプット）:** 人事データベースからデータをインポートしてプレート作成は可能か？
3. **汎用性:** 人員配置以外のホワイトボード用途への展開可能性は？
4. **比較検討:** より汎用性の高い代替ツールとの比較検討

→ これらが不明瞭な状態での導入は**リスクが高い**と判断
