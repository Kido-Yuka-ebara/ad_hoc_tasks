# 成果物0-2: フォークフローに関する所感

## 1. ツールの位置づけ

- **簡易版としての差別化:** ロジタンフォークと差別化するため、導入容易性を重視し機能を制限
- **取得データの範囲:** フォークの「稼働状態」のみ（位置情報は含まない）

## 2. 主な懸念事項

### 2.1 稼働データ取得の意義への疑問

- **稼働を取ること自体の意味:** フォークの稼働を取ること自体にあまり意味はない
- **非稼働フォークの削減リスク:**
  - 非稼働のフォークが多かったとしても、移動時間がある以上、本当に減らしても工場の稼働に問題がないか保証されない
  - 工場の稼働が平準化されておらず、フォーク運搬にムラがある場合、むやみに減らすとコアタイムに対応できなくなる恐れ

### 2.2 本質的な改善の順序

- **運搬の本質:** あくまで運搬は製造ラインの後工程引き取りに引っ張られる形で実践されるべき
- **優先すべきこと:** まずはそこのルール作りから着手すべき

→ フォーク台数適正化は、**ルール整備の後に検討すべき課題**

## 3. ツールの適用シーン

### 3.1 限定的な用途

| 用途 | 評価 | コメント |
| :--- | :--- | :--- |
| **台数適正化の優先順位づけ** | △ | 工程着手の優先順位づけ程度にはなる |
| **即効性のある改善** | × | 導入すればすぐに何かが良くなるものではない |
| **お試し導入** | ○ | 着手が億劫でまずは稼働を見たいなら、簡易的で金額もそこまでではないのでお試しには良い |
| **永続利用** | × | 少なくとも永続して使うものではないかもしれない |

### 3.2 唯一の明確なメリット: 安全面

- **急停止・急発進の検知:** 安全という意味では良い
- **活用例:** 人車分離を施策にする説得材料になる程度

→ **安全面以外の明確なメリットが見出しにくい**

## 4. 総合所見

| 評価項目 | 評価 | コメント |
| :--- | :--- | :--- |
| **データの意義** | × | 稼働状態取得自体の意味が不明瞭 |
| **台数適正化** | △ | 優先順位づけ程度、即効性なし |
| **導入容易性** | ○ | 簡易的で金額も手頃 |
| **お試し利用** | ○ | まずは稼働を見たいなら有効 |
| **永続利用** | × | 長期利用の価値は低い |
| **安全面** | ○ | 人車分離の説得材料として活用可 |

## 5. 導入検討の判断基準

### 導入を検討してもよいケース

1. **製造ラインのルールが整備済み:** 後工程引き取りのルールが確立されている
2. **優先順位づけのため:** 台数適正化の工程着手の優先順位づけが目的
3. **安全施策の根拠:** 人車分離などの安全施策の説得材料が必要
4. **お試し導入:** まずは稼働状況を可視化してみたい（短期利用前提）

### 導入を見送るべきケース

1. **即効性を期待:** 導入すればすぐに改善効果が出ると期待している
2. **永続利用前提:** 長期的に利用し続けることを前提としている
3. **ルール未整備:** 製造ラインの後工程引き取りルールが未整備

## 6. 推奨される検討順序

1. **第一優先:** 製造ラインの後工程引き取りルールの整備
2. **第二優先:** ルール整備後、フォーク台数適正化の優先順位づけ
3. **第三優先:** お試しとしてフォークフロー導入（短期利用）
4. **安全施策:** 並行して安全面のデータ活用（人車分離の説得材料）

→ **本質的な改善の順序を守らずに導入してもROIは低い**
