# 業務成熟度マトリクス（デジボ）

**📄 この資料について:**  
デジボが活用できる業務として、製造ライン人員配置管理業務をピックアップ。  
Level 0（属人化）からLevel 7（AI化）の業務状態を定義し、各レベルで必要な施策・ツールを体系化。  
デジボはLevel 4（可視化）で必要な定常ツールとして位置づけられる。  
次の資料「前提条件体系化」でデジボ導入の具体的前提条件を確認。

---

## 評価対象業務

**業務**: 製造ライン人員配置管理業務

**業務KPI（全レベル共通）:**
- 配置計画作成時間（分）
- 配置変更対応時間（分）
- スキルマッチング精度（%）

---

## 業務成熟度マトリクス

※ ✓ = 当部門担当

| レベル | 状態定義 | 業務の状態 | レベル到達の非定常施策・投資 | レベル維持の定常施策・投資 |
|:---:|:---|:---|:---|:---|
| **Level 0** | **プロセス未体系化** | リーダーの経験と勘による口頭指示。配置記録なし | （Level 0は記載なし） | - 口頭指示 |
| **Level 1** | **プロセス体系化** | 配置手順の文書化。配置図の作成。配置ルールの明文化 | - ✓ 業務プロセスの可視化・文書化<br>- ✓ 配置ルールの暗黙知の抽出 | - 手順書<br>- 配置図 |
| **Level 2** | **手順標準化** | SOPに基づく配置計画。作業者リストとスキルの電子管理。配置状況の物理的可視化 | - ✓ SOP作成<br>- 作業者スキル一覧表の整備 | - SOP<br>- Excel管理表<br>- ホワイトボード |
| **Level 3** | **手順遵守** | SOP通りの配置の徹底。配置変更時の必須記録・報告。定期的な実績振り返り | - 手順遵守の仕組み構築<br>- 逸脱時の記録・報告ルール整備 | - 記録・報告の仕組み<br>- 定期レビュー会議 |
| **Level 4** | **可視化** | 配置変更の即時記録。スキル情報の即時更新。配置状況の常時最新可視化 | - ✓ デジタル配置管理ツール導入<br>- ✓ リアルタイム記録の仕組み構築 | - ✓ **<font color="red">デジタル配置管理ツール（デジボ等）</font>**<br>- ✓ 基幹システム<br>- モニター・PC |
| **Level 5** | **最適化** | 蓄積データに基づく配置パターン最適化。計画的な多能工化推進。KPI管理による継続改善 | - ✓ データ分析体制の構築<br>- ✓ 改善PDCAサイクルの確立 | - ✓ データ分析ツール<br>- ✓ KPIダッシュボード |
| **Level 6** | **省人化** | 配置案の自動生成。リーダーによる確認・微調整のみ。配置業務工数の大幅削減 | - ✓ 配置シミュレーション機能の開発<br>- ✓ AI/最適化アルゴリズムの検討 | - ✓ 配置最適化システム<br>- ✓ 自動生成機能 |
| **Level 7** | **人排除orAI化（ありたい姿）** | 需要予測・スキルマッチング・配置最適化の自動実行。人による例外対応のみ | - ✓ 完全自動化・AI化の検討<br>- ✓ リアルタイム最適配置の実現<br>- ✓ 組織変革・人材育成 | - ✓ AI配置システム<br>- ✓ 自動実行基盤 |
