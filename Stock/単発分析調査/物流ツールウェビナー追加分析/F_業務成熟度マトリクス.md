# 業務成熟度マトリクス（フォークフロー）

## 評価対象業務

**業務**: フォークリフト運用管理業務

**業務KPI（全レベル共通）:**
- 稼働率（%）
- から走行率（%）- 低いほど良い
- 危険運転発生率（回/時間）- 低いほど良い

---

## 業務成熟度マトリクス

※ ✓ = 当部門担当

| レベル | 状態定義 | 業務の状態 | レベル到達の非定常施策・投資 | レベル維持の定常施策・投資 |
|:---:|:---|:---|:---|:---|
| **Level 0** | **プロセス未体系化** | 運転者の裁量で運用。稼働状況は不明。適正台数も不明 | （Level 0は記載なし） | - フォークリフト（車両）<br>- 運転免許保有者<br>- 基本的な安全ルール<br>- 安全教育（初回のみ） |
| **Level 1** | **プロセス体系化** | 運行ルールの文書化。配車計画の紙での作成。日報での稼働記録 | - ✓ 運用実態の把握<br>- ✓ 運用ルールの文書化 | - 運行ルール文書<br>- 配車計画表（紙/Excel）<br>- 運転日報フォーマット |
| **Level 2** | **手順標準化** | SOPに基づく運行。配車の電子管理。日報集計での月次報告 | - ✓ SOP作成<br>- 運行記録の標準化 | - 運行SOP<br>- 電子配車管理表（Excel等）<br>- 運転日報<br>- 月次集計レポート |
| **Level 3** | **手順遵守** | SOP通りの運行が徹底。日報記入漏れなし。ヒヤリハット報告の仕組みあり | - ✓ 運用実態の把握（✓ **<font color="red">稼働計測ツール等での一時計測</font>**）<br>- ✓ ルール遵守状況の検証<br>- 日報記入の徹底ルール | - 日報提出チェック体制<br>- ヒヤリハット報告制度<br>- 定期安全会議 |
| **Level 4** | **可視化** | 配車指示の電子化。指示ベースでの稼働状況把握。計画と実績の差異管理 | - ✓ シミュレーションモデルの構築 | - ✓ 配車指示管理システム<br>- 指示実績記録<br>- ✓ 稼働シミュレーションツール<br>- ✓ データ分析ツール |
| **Level 5** | **最適化** | 稼働データに基づく台数最適化。動線改善でから走行削減。安全教育を定量データで実施 | - ✓ データ活用体制の構築<br>- ✓ 改善PDCAサイクルの確立<br>- ✓ 適正台数の算出<br>- ✓ 改善効果の検証（✓ 稼働計測ツール等での一時計測） | - ✓ データ分析体制<br>- ✓ データ分析スキル・知識<br>- 改善会議体<br>- ✓ データ駆動型安全教育 |
| **Level 6** | **省人化** | 最適配車の自動提案。リアルタイム位置情報による効率的指示。管理工数の大幅削減 | - ✓ リアルタイム監視システムの検討<br>- ✓ 配車最適化アルゴリズムの開発 | - ✓ 配車最適化システム<br>- ✓ リアルタイム位置情報システム<br>- ✓ 自動配車アルゴリズム |
| **Level 7** | **人排除orAI化（ありたい姿）** | 搬送の自動化と無人実行。人による監視・例外対応のみ | - ✓ 自動運転・無人搬送の検討<br>- 自動搬送車両への移行検討<br>- ✓ 組織変革・人材育成 | - 自動運転搬送車両<br>- ✓ 自律運行システム<br>- ✓ 統合管理プラットフォーム<br>- 遠隔監視体制 |
